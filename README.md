 Краткая характеристика проекта:
1.  Целевой потребитель проекта - разработчики сайта. Разработчики должны иметь возможность, используя автоматизированое 
тестовое покрытие, видеть как  вносимые программные изменения отображаются на работоспособности, информативности и 
внешнем виде сайта.
2. Пользовательская концепция проекта: пользователь (прежде всего разработчиики сайта) хочет иметь в своем распоряжении
тестирующую машину, которая в любой момент времени покажет:
- где "сломался" сайт в ходе доработки,  
-  максимальнольно информативно выдаст информацию в чем состоит проблема.
3. Основные задачи проекта:
Задача 1: определить оптимальные для поиска элементов сайта локаторы:
__ для статичных элементов. В том числе для заголовков и сообщений,
__для динамичнных элементов. В том числе для кнопок перемещающих:
       - в другие разделы главной страницы (тот же URL),
       - на другие страницы сайта (другой URL)
Задача 2:  отобрать из числа локаторов наиболее  приемлемые, т.е.те которые с наибольшей вероятностью не будут меняться
при доработке сайта и взять их за осннову при разработке тестов.
Задача 3: составить UI тесты для следующих групп элементов страницы:
   - для загрузки сайта через VPN, 
   - для кнопок осуществляющих навигацию по главнйо странице,
   - для кнопок осуществляющих перемещние пользователя на другие страницы сайта (проверить кликабельность кнопок),
- для проверки  доступности URL всех страниц привязанных к главной странице сайта
Задача 4: создать dockerfile, который будет использован для сборки и запуска тестов.
Задача 5: осуществить заливку проекта на GitHab
